Gelöste technische Herausforderungen

4-Sprachen-i18n-System von Grund auf — ohne externe Bibliothek

Kein i18next, keine externe Abhängigkeit. Jeder übersetzbare Text auf jeder Seite ist mit einem data-i18n-Attribut versehen, und die i18n-Engine durchläuft den gesamten DOM und tauscht Texte beim Sprachenwechsel live aus. Die Herausforderung: Inputs benötigen .value, Platzhalter benötigen .placeholder, und alles andere nutzt .textContent — jedes separat behandelt. 471 Zeilen Übersetzungsschlüssel in vier Sprachen. Standardsprache ist Georgisch — ein klares Signal, wer die eigentliche Zielgruppe ist: die georgischsprachige Diaspora in Mitteleuropa.

WhatsApp Business Cloud API mit Graceful Fallback

Das Backend integriert Metas Graph API (v18.0), um bei jeder Buchung eine WhatsApp-Nachricht an den Geschäftsinhaber zu senden. Aber Produktions-APIs versagen — Credentials laufen ab, Rate Limits werden überschritten, Netzwerkprobleme treten auf. Anstatt einen WhatsApp-Fehler die Buchung zum Absturz zu bringen, greift der Dienst still auf einen Click-to-Chat-Link zurück, wenn Credentials nicht konfiguriert sind oder der API-Aufruf fehlschlägt. Alle drei Benachrichtigungskanäle (Admin-E-Mail, Kunden-E-Mail, WhatsApp) laufen parallel über Promise.allSettled() — die Buchung liefert immer Erfolg, auch wenn ein Kanal ausfällt.

Promise.allSettled() — Eine Buchung, die kein Nebeneffekt zum Scheitern bringen kann

Der naive Ansatz ist Promise.all() — aber das bedeutet: ein fehlgeschlagener WhatsApp-Aufruf bringt die gesamte Buchungstransaktion zum Absturz. Der Kunde sendet sein Formular, etwas geht in einer Drittanbieter-API schief, und er sieht einen Fehler. Seine Buchung ist verloren. allSettled() sorgt dafür, dass jeder Benachrichtigungsdienst sein Ergebnis unabhängig protokolliert. Die Kernlogik der Buchung schreibt immer Erfolg. Fehler jedes Kanals werden protokolliert, nicht weitergeleitet.

Multer-Upload → E-Mail-Anhang-Pipeline

Buchungsfotos durchlaufen eine vollständige Pipeline: Multer empfängt sie auf HTTP-Ebene, speichert sie mit kollisionssicheren eindeutigen Dateinamen auf der Festplatte, und anschließend liest der Controller sie zurück und hängt sie als echte E-Mail-Anhänge — keine Links — an die Admin-Benachrichtigung an. Dieselben Dateipfade werden auch in die WhatsApp-Nachricht serialisiert, damit der Inhaber direkt in WhatsApp sieht, welche Möbel transportiert werden sollen. Die Koordinierung von Multer-Middleware, Buchungscontroller und zwei separaten Benachrichtigungsdiensten ohne Dateiverlust oder doppelte Pfade erforderte sorgfältige Reihenfolge.

Bot-Schutz ohne CAPTCHA

reCAPTCHA verursacht UX-Aufwand — ein zusätzlicher Klick, manchmal ein Rätsel, immer eine Anfrage an Google. Das Buchungsformular setzt stattdessen auf ein Honeypot-Feld: ein verstecktes website-Eingabefeld, das echte Nutzer nie sehen und nie ausfüllen. Bots, die blind alle Formularfelder befüllen, tragen es ein. Der serverseitige Validator in validation.js prüft es und lehnt die Einreichung still ab. Echte Nutzer erfahren keinerlei Aufwand; Bot-Einreichungen erreichen die Benachrichtigungs-Pipeline nie.

public/js/i18n.js — DOM walker for live language switching JavaScript 
// Zero-dependency i18n: walks the DOM, swaps every translatable node
function applyTranslations(lang) {
  const t = translations[lang];
  if (!t) return;

  // Text content — covers most elements (headings, labels, paragraphs)
  document.querySelectorAll('[data-i18n]').forEach(el => {
    const key = el.dataset.i18n;
    if (t[key]) el.textContent = t[key];
  });

  // Placeholders — inputs and textareas need a separate pass
  document.querySelectorAll('[data-i18n-placeholder]').forEach(el => {
    const key = el.dataset.i18nPlaceholder;
    if (t[key]) el.placeholder = t[key];
  });

  // Input values (e.g. submit buttons rendered as <input type="submit">)
  document.querySelectorAll('[data-i18n-value]').forEach(el => {
    const key = el.dataset.i18nValue;
    if (t[key]) el.value = t[key];
  });

  localStorage.setItem('preferredLanguage', lang);
  document.documentElement.lang = lang;
}

src/controllers/bookingController.js — Parallel notifications with allSettled Node.js 
// allSettled — each notification channel fails independently
// The booking ALWAYS returns success to the user
const results = await Promise.allSettled([
  emailService.sendAdminNotification(bookingData, uploadedFiles),
  emailService.sendCustomerConfirmation(bookingData),
  whatsappService.sendBookingNotification(bookingData),
]);

// Log each channel result independently — no silent failures
results.forEach((result, index) => {
  const channels = ['Admin email', 'Customer email', 'WhatsApp'];
  if (result.status === 'fulfilled') {
    console.log(`[Booking] ${channels[index]}: sent`);
  } else {
    console.error(`[Booking] ${channels[index]} failed:`, result.reason);
  }
});

// Always return 200 — notification failures are not booking failures
res.status(200).json({ success: true, message: 'Booking received' });
src/services/whatsappService.js — Meta Graph API with fallback Node.js 
async function sendBookingNotification(bookingData) {
  // No credentials? Return a click-to-chat link instead of throwing
  if (!process.env.WHATSAPP_TOKEN || !process.env.WHATSAPP_PHONE_ID) {
    return {
      fallback: true,
      link: `https://wa.me/${process.env.BUSINESS_PHONE}?text=New booking`
    };
  }

  const response = await axios.post(
    `https://graph.facebook.com/v18.0/${process.env.WHATSAPP_PHONE_ID}/messages`,
    {
      messaging_product: 'whatsapp',
      to: process.env.BUSINESS_PHONE,
      type: 'text',
      text: { body: formatBookingMessage(bookingData) },
    },
    { headers: { Authorization: `Bearer ${process.env.WHATSAPP_TOKEN}` } }
  );

  return response.data;
}

// Honeypot: hidden input named "website" — real users never fill it
// Bots that blindly complete all fields get silently rejected
const validateBooking = [
  body('website')
    .isEmpty()
    .withMessage('Bot detected'),

  body('name')
    .trim().notEmpty().withMessage('Name is required')
    .isLength({ max: 100 }),

  body('phone')
    .trim().notEmpty().withMessage('Phone is required')
    .isMobilePhone(),

  (req, res, next) => {
    const errors = validationResult(req);
    if (!errors.isEmpty()) {
      return res.status(400).json({ errors: errors.array() });
    }
    next();
  }
