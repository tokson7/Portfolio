Technical Challenges Solved

4-Language i18n System Built from Scratch — No Library

No i18next, no external dependency. Every translatable string on every page is tagged with a data-i18n attribute, and the i18n engine walks the entire DOM and swaps text live on language switch. The tricky part: inputs need .value, placeholders need .placeholder, and everything else uses .textContent — each handled separately. 471 lines of translation keys across four languages. Default language is Georgian — a direct signal of who the real user base is: the Georgian-speaking diaspora in Central Europe.

WhatsApp Business Cloud API with Graceful Fallback

The backend integrates with Meta's Graph API (v18.0) to send WhatsApp messages to the business owner on every booking. But production APIs fail — credentials expire, rate limits hit, network issues happen. Rather than letting a WhatsApp failure crash the booking, the service silently falls back to a click-to-chat link when credentials are unconfigured or the API call fails. All three notification channels (admin email, customer email, WhatsApp) run in parallel using Promise.allSettled() — the booking always returns success, even if one channel drops.

Promise.allSettled() — A Booking That Can't Be Killed by a Side Effect

The naive approach is Promise.all() — but that means one failed WhatsApp call kills the entire booking transaction. The customer submits their form, something goes wrong in a third-party API, and they see an error. Their booking is lost. Using allSettled() instead means every notification service logs its own result independently. The core booking logic always succeeds. Each channel's failure is logged, not propagated.

Multer Upload → Email Attachment Pipeline

Booking photos travel a full pipeline: Multer receives them at the HTTP layer, stores them on disk with collision-safe unique filenames, and then the controller reads them back and attaches them as real email attachments — not links — to the admin notification. The same file paths are also serialized into the WhatsApp message text so the owner can see what furniture they're being asked to move, directly in WhatsApp. Coordinating Multer middleware, the booking controller, and two separate notification services without dropping files or duplicating paths required careful ordering.

Bot Protection Without CAPTCHA

reCAPTCHA adds UX friction — extra click, sometimes a puzzle, always a request to Google. The booking form instead uses a honeypot: a hidden website input field that real users never see and never fill. Bots that blindly complete all form fields populate it. The server-side validator in validation.js checks it and silently rejects the submission. Real users experience zero friction; bot submissions never reach the notification pipeline.

public/js/i18n.js — DOM walker for live language switching JavaScript 
// Zero-dependency i18n: walks the DOM, swaps every translatable node
function applyTranslations(lang) {
  const t = translations[lang];
  if (!t) return;

  // Text content — covers most elements (headings, labels, paragraphs)
  document.querySelectorAll('[data-i18n]').forEach(el => {
    const key = el.dataset.i18n;
    if (t[key]) el.textContent = t[key];
  });

  // Placeholders — inputs and textareas need a separate pass
  document.querySelectorAll('[data-i18n-placeholder]').forEach(el => {
    const key = el.dataset.i18nPlaceholder;
    if (t[key]) el.placeholder = t[key];
  });

  // Input values (e.g. submit buttons rendered as <input type="submit">)
  document.querySelectorAll('[data-i18n-value]').forEach(el => {
    const key = el.dataset.i18nValue;
    if (t[key]) el.value = t[key];
  });

  localStorage.setItem('preferredLanguage', lang);
  document.documentElement.lang = lang;
}

src/controllers/bookingController.js — Parallel notifications with allSettled Node.js 
// allSettled — each notification channel fails independently
// The booking ALWAYS returns success to the user
const results = await Promise.allSettled([
  emailService.sendAdminNotification(bookingData, uploadedFiles),
  emailService.sendCustomerConfirmation(bookingData),
  whatsappService.sendBookingNotification(bookingData),
]);

// Log each channel result independently — no silent failures
results.forEach((result, index) => {
  const channels = ['Admin email', 'Customer email', 'WhatsApp'];
  if (result.status === 'fulfilled') {
    console.log(`[Booking] ${channels[index]}: sent`);
  } else {
    console.error(`[Booking] ${channels[index]} failed:`, result.reason);
  }
});

// Always return 200 — notification failures are not booking failures
res.status(200).json({ success: true, message: 'Booking received' });
src/services/whatsappService.js — Meta Graph API with fallback Node.js 
async function sendBookingNotification(bookingData) {
  // No credentials? Return a click-to-chat link instead of throwing
  if (!process.env.WHATSAPP_TOKEN || !process.env.WHATSAPP_PHONE_ID) {
    return {
      fallback: true,
      link: `https://wa.me/${process.env.BUSINESS_PHONE}?text=New booking`
    };
  }

  const response = await axios.post(
    `https://graph.facebook.com/v18.0/${process.env.WHATSAPP_PHONE_ID}/messages`,
    {
      messaging_product: 'whatsapp',
      to: process.env.BUSINESS_PHONE,
      type: 'text',
      text: { body: formatBookingMessage(bookingData) },
    },
    { headers: { Authorization: `Bearer ${process.env.WHATSAPP_TOKEN}` } }
  );

  return response.data;
}

  body('website')
    .isEmpty()
    .withMessage('Bot detected'),

  body('name')
    .trim().notEmpty().withMessage('Name is required')
    .isLength({ max: 100 }),

  body('phone')
    .trim().notEmpty().withMessage('Phone is required')
    .isMobilePhone(),

  (req, res, next) => {
    const errors = validationResult(req);
