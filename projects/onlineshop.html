Gelöste technische Herausforderungen

Echtzeit-Benachrichtigungsmotor mit Dauerton

Entwicklung eines dualen Systems — Pusher WebSocket für sofortige Benachrichtigungen mit Dauerton, Polling als Fallback. Der Admin hört persistenten Ton, bis die Bestellung angenommen wird. Kritische Anforderung: verpasste Bestellung = entgangener Umsatz.

Duales Zahlungsgateway — Manueller Zahlungseinzug (Stripe + PayPal)

Die Karte wird beim Checkout autorisiert (gehalten), Mittel werden erst eingezogen, wenn der Admin akzeptiert. Bei Ablehnung wird die Reservierung automatisch freigegeben. Erforderte die Handhabung von Webhooks, Ablauf der Autorisierung (7 Tage Stripe / 3 Tage PayPal) und Sonderfälle.

Vercel CPU-Erschöpfung — 95,6% Polling-Reduktion

Produktion überschritt Vercels 4h/Monat CPU-Limit durch 2-Sekunden-Polling (1.800 API-Aufrufe/Stunde). Diagnose via Log-Analyse, Umstieg auf Pusher-Events, Polling auf 5-Minuten-Intervalle reduziert (12 Aufrufe/Stunde). Kein Einfluss auf die Nutzererfahrung.

Gast → Registrierter Kunde Migrationsfehler

Wiederkehrende Kunden konnten keine Bestellungen aufgeben — Prisma-Unique-Constraint-Verletzung bei E-Mail. Gelöst mit 4-Schritt-Kundenabfrage: per accountId → per E-Mail → bestehenden Gastdatensatz verknüpfen → neu erstellen. Ermöglicht nahtlose Gast-zu-Konto-Migration.

Skalierbare Massen-E-Mail-Warteschlange

Resends kostenloses Kontingent: 2 E-Mails/Sekunde. Ursprüngliche Implementierung sendete 10 gleichzeitig → 80% Fehlerrate. Neugestaltung als Hintergrundwarteschlange — Admin löst Versand aus, API antwortet sofort, Cron verarbeitet 400 E-Mails/Minute mit Wiederholungslogik und Fortschrittsverfolgung.

// Real-time order notification — looping sound engine
export function startSoundLoop(soundName: string, volume: number): void {
  if (audioContext?.state === 'suspended') {
    audioContext.resume();
  }

  const audio = new Audio(`/sounds/${soundName}.mp3`);
  audio.volume = Math.min(Math.max(volume / 100, 0), 1);
  audio.loop = true;

  audio.play().then(() => {
    currentLoopingAudio = audio;
    isLooping = true;
  }).catch(err => {
    console.error('[NotificationEngine] Playback failed:', err);
  });
}

export function stopSoundLoop(): void {
  if (currentLoopingAudio) {
    currentLoopingAudio.pause();
    currentLoopingAudio.currentTime = 0;
    currentLoopingAudio = null;
    isLooping = false;
  }
}
src/app/api/orders/route.ts — Guest-to-registered customer migration TypeScript 
// 4-step customer lookup — prevents unique constraint violations
let customerRecord = await prisma.customer.findFirst({
  where: { customerAccountId: customer.id },
});

if (!customerRecord && customer.email) {
  customerRecord = await prisma.customer.findFirst({
    where: { email: customer.email },
  });

  // Link existing guest customer to registered account
  if (customerRecord && !customerRecord.customerAccountId) {
    await prisma.customer.update({
      where: { id: customerRecord.id },
      data: { customerAccountId: customer.id },
    });
  }
}

if (!customerRecord) {
  customerRecord = await prisma.customer.create({
    data: {
      name: customerName,
      email: customer.email,
      customerAccountId: customer.id,
    },
  });
}

prisma/schema.prisma — Order model Prisma 
model Order {
  id           String      @id @default(cuid())
  orderNumber  String      @unique
  status       OrderStatus @default(PENDING)
  type         OrderType   @default(TAKEAWAY)

  // Payment — supports Stripe + PayPal + Cash
  paymentMethod           PaymentMethod?
  paymentStatus           PaymentStatus  @default(PENDING)
  stripePaymentIntentId   String?   @unique
  paypalOrderId           String?   @unique
  paypalAuthorizationId   String?
  paymentAuthorizedAt     DateTime?
  paymentCapturedAt       DateTime?
  authorizationExpiry     DateTime?

  // Vienna district-based minimum order enforcement
  minimumOrderAdjustment  Float     @default(0)
  minimumOrderThreshold   Float?
  deliveryPostalCode      String?
  deliveryDistrict        String?

  // Two-step customer cancel request
  cancelRequestStatus  String?  // "REQUESTED" → "APPROVED" / "REJECTED"

  // Timestamps
  createdAt   DateTime  @default(now())
  completedAt DateTime?

  @@index([status])
  @@index([paymentStatus])
}
